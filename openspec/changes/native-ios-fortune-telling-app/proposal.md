## Why

当前项目需要从空白原生 iOS 工程演进为可交付的算命类应用，核心价值来自用户输入生日、拍摄掌纹/面相、选择黄道吉日后，由服务端大模型生成结构化命理结果。先通过 OpenSpec 固化产品边界、端云职责和实现任务，可避免直接实现时把推理逻辑、隐私授权、拍摄流程和结果展示耦合在一起。

## What Changes

- 新增纯原生 iOS 应用主流程，覆盖首页入口、生日命理、掌纹面相、黄道吉日和历史结果。
- 新增生日命理能力：用户输入公历生日、出生时间、出生地区和性别等必要字段后，服务端大模型返回星座、生辰八字、五行、性格、事业、感情、健康等结构化内容。
- 新增掌纹与面相能力：用户通过相机或相册提交掌纹、面部照片，客户端完成授权、采集、裁剪和上传，服务端大模型返回掌纹命理与面相信息。
- 新增黄道吉日能力：用户选择目标事项、候选日期范围和个人生日信息后，服务端大模型返回吉日推荐、避忌事项和推荐理由。
- 新增端云协议边界：客户端只负责采集、校验、展示、状态管理和错误处理，所有命理内容生成由服务端大模型驱动。
- 新增基础隐私与安全约束：生日、面部、掌纹等敏感信息需要用户授权、传输加密、上传状态提示和可删除的历史记录。

## Capabilities

### New Capabilities

- `birthday-fortune`: 生日输入与星座、生辰八字、五行等命理结果展示。
- `palm-face-reading`: 掌纹与面相拍摄、上传、分析结果展示。
- `auspicious-date-selection`: 黄道吉日事项选择、日期范围输入、吉日推荐结果展示。
- `server-driven-ai-results`: 服务端大模型驱动的统一请求、任务状态、结果解析和错误处理协议。
- `privacy-and-history`: 敏感权限提示、用户数据删除、分析历史保存与复查。

### Modified Capabilities

- 无。

## Impact

- 影响 iOS 原生应用结构：SwiftUI 页面、导航、表单、相机/相册能力、异步任务状态和结果展示组件。
- 需要新增服务端 API 契约：生日命理、掌纹面相、黄道吉日、任务查询、历史记录与删除接口。
- 需要新增系统权限与配置：相机、相册、网络访问，以及面部/掌纹图片采集的隐私说明。
- 需要新增本地数据与缓存策略：分析历史、任务状态、失败重试和用户主动删除。
- 后续实现需补充真实接口地址、鉴权方式、服务端返回 schema、错误码和埋点策略。
